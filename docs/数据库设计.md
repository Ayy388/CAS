### 2.3 关键设计决策

**设计决策一：不需要 campaign_offering 中间表**

- **证据**：campaign 与 offering 都通过 semester_id 关联到同一学期。PRD 未要求某个活动只包含部分开课。查询时通过 `WHERE offering.semester_id = campaign.semester_id` 即可关联。
- **V2.0 扩展**：如果需要"部分开课参与活动"，增加 campaign_offering 表，不影响现有表结构。

**设计决策二：enrolled_count 冗余字段**

- **证据**：课程大厅列表（页面原型 2.2）需要展示每门课程的剩余名额。如果每张卡片都 COUNT enrollment 表，30 门课需要 30 次子查询。冗余字段一次 JOIN 即可取得。
- **同步方式**：原子 UPDATE `SET enrolled_count = enrolled_count + 1 WHERE id = ? AND enrolled_count < max_capacity`

**设计决策三：枚举用 VARCHAR 不用 ENUM**

- **证据**：MySQL ENUM 类型在以下场景表现不佳：1) 增加新值需 ALTER TABLE；2) ORDER BY 按定义顺序而非字母序；3) 跨数据库迁移困难。VARCHAR + 应用程序枚举更灵活。

### 3.6 enrollment — 选课记录表

**用途**：记录学生选课行为，是系统的核心业务表。

**字段定义**：

| 字段名 | 类型 | 约束 | 说明 | 对应前端类型 |
|--------|------|------|------|------------|

| offering_id | BIGINT | FK, NOT NULL | 所选开课 → course_offering.id | Enrollment.offeringId |
| student_id | BIGINT | FK, NOT NULL | 学生 → sys_user.id | Enrollment.studentId |
| status | VARCHAR(10) | NOT NULL DEFAULT 'ENROLLED' | ENROLLED / APPROVED / REJECTED / DROPPED | Enrollment.status |
| enrolled_at | DATETIME | NOT NULL | 报名时间 | Enrollment.enrolledAt |
| reviewed_at | DATETIME | | 审核时间 | Enrollment.reviewedAt |
| created_at | DATETIME | NOT NULL | | |
| updated_at | DATETIME | NOT NULL | | |

**外键**：
- `fk_enrollment_campaign` → selection_campaign(id)
- `fk_enrollment_offering` → course_offering(id)
- `fk_enrollment_student` → sys_user(id)

**唯一约束与索引**：
- **`uq_enrollment_campaign_student`** UNIQUE (campaign_id, student_id) — 核心约束，保证 BR-03
- `idx_enrollment_offering` (offering_id) — 查询某门课程的报名学生列表
- `idx_enrollment_student` (student_id) — 学生查看自己的选课记录
- `idx_enrollment_enrolled_at` (enrolled_at) — 抢课趋势图按分钟聚合（数据看板需求）
- `idx_enrollment_status` (status) — 按状态过滤

**设计依据**：

| 设计决策 | 理由 | 证据 |
|---------|------|------|
| UNIQUE (campaign_id, student_id) | BR-03：一个活动内一名学生只能有一条选课记录 | PRD BR-03 |
| 退课用 DELETE 而非标记 DROPPED | 退课后学生可重新选课，DELETE 后 UNIQUE 约束不阻塞重新插入 | 页面原型 2.4 "退课" |

> **关于退课的详细说明**：当学生退课时，我们 DELETE 该 enrollment 记录并原子 DECREMENT enrolled_count。如果学生立即重新选同一门或其他课程（名额允许），新的 INSERT 不会违反 UNIQUE 约束。这样设计最简洁，不需要软删除或复合唯一索引。对于 V1.0 MVP，这个方案足够。

## 四、并发抢课方案

### 4.1 核心思路

采用 **原子 UPDATE + 检查影响行数** 替代显式行锁，避免死锁和连接池耗尽。

### 4.2 抢课流程

```
请求抢课
   │
   ▼
① 校验 campaign.status = 'ACTIVE'
   │
   ▼
② 校验该 student 在该 campaign 下无 ENROLLED/APPROVED 记录
   │ (BR-03：一个活动仅允许选一门课)
   ▼
③ 校验 student.grade/major 匹配 offering.open_grade/open_major
   │ (BR-05：开放范围规则)
   ▼
④ 原子 UPDATE：
   UPDATE course_offering
      SET enrolled_count = enrolled_count + 1
    WHERE id = ? AND enrolled_count < max_capacity
   │
   ├── affected_rows = 0 → 返回"已满员"
   │
   ▼
⑤ INSERT INTO enrollment (...)
   │
   ▼
⑥ 返回"抢课成功"
```

**事务隔离级别**：READ_COMMITTED（InnoDB 默认），无需 `SELECT ... FOR UPDATE`

### 4.3 退课流程

```
请求退课
   │
   ▼
① 校验 campaign 未结束（status = ACTIVE 或 PENDING）
   │
   ▼
② 原子 DECREMENT：
   UPDATE course_offering
      SET enrolled_count = enrolled_count - 1
    WHERE id = ? AND enrolled_count > 0
   │
   ▼
③ DELETE FROM enrollment WHERE id = ?
   │
   ▼
④ 返回"退课成功"
```

### 4.4 为什么不使用行锁

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| SELECT ... FOR UPDATE | 直观 | 持有锁期间所有并发请求排队，连接池快速耗尽 | ❌ 不推荐 |
| 原子 UPDATE + affected_rows | 无锁，连接池友好 | 需要检查 affected_rows | ✅ 选用 |
| Redis 分布式锁 | 适合极高并发 | 引入额外组件和运维成本 | V2.0 按需引入 |

**证据**：高校选课场景（数千学生级别），一门课 50 个名额。原子 UPDATE 的冲突概率很低。即使多人同时抢最后一个名额，InnoDB 的行级锁在 UPDATE 上自动生效，只有一个会成功。

## 五、审核与通知流程

### 5.1 开课审核

**触发时机**：选课活动结束后（campaign.status = 'ENDED'）

**审核流程**：

```
① 查询所有 course_offering WHERE semester_id = ? AND status = 'PENDING'
   │
   ▼
② 对每条 offering，计算建议结果：
   ├── enrolled_count >= min_enrollment → "建议开课"
   └── enrolled_count <  min_enrollment → "建议取消"
   │
   ▼
③ 管理员逐条审核：
   ├── 审核通过 → UPDATE offering.status = 'APPROVED'
   │               → UPDATE enrollment SET status = 'APPROVED'
   │               → INSERT notification (type = 'APPROVED')
   │
   └── 审核不通过 → UPDATE offering.status = 'REJECTED'
                    → UPDATE enrollment SET status = 'REJECTED'
                    → INSERT notification (type = 'REJECTED')
```

### 5.2 批量通知生成 SQL

```sql
-- 审核通过后，为所有报名学生生成通知
-- 注意：此 SQL 在 MQ 消费者中执行，此时 enrollment.status 已被事务更新为 APPROVED
-- 因此不按 status 过滤，而是按 offering_id 查询所有选课记录
INSERT INTO notification (user_id, type, title, content, course_name, is_read, created_at)
SELECT
    e.student_id,
    'APPROVED',
    CONCAT('《', c.name, '》审核通过'),
    CONCAT('您报名的《', c.name, '》已审核通过，将正常开课。'),
    c.name,
    0,
    NOW()
FROM enrollment e
JOIN course_offering co ON e.offering_id = co.id
JOIN course c ON co.course_id = c.id
WHERE e.offering_id = ?;
```

## 六、数据看板查询

### 6.1 KPI 统计

```sql
-- 总课程数（页面原型 4.6）
SELECT COUNT(*) AS total FROM course;

-- 总教师数
SELECT COUNT(*) AS total FROM sys_user WHERE role = 'TEACHER';

-- 总学生数（仅统计启用状态）
SELECT COUNT(*) AS total FROM sys_user WHERE role = 'STUDENT' AND status = 1;

-- 总报名人数
SELECT COUNT(*) AS total FROM enrollment;
```

### 6.2 热门课程 TOP10

```sql
SELECT
    c.name          AS course_name,
    co.id           AS offering_id,
    COUNT(e.id)     AS enrollment_count
FROM course_offering co
JOIN course c ON co.course_id = c.id
LEFT JOIN enrollment e ON e.offering_id = co.id
WHERE co.semester_id = ?
GROUP BY co.id, c.name
ORDER BY enrollment_count DESC
LIMIT 10;
```

### 6.3 抢课趋势（按分钟）

```sql
SELECT
    DATE_FORMAT(enrolled_at, '%Y-%m-%d %H:%i') AS minute,
    COUNT(*)                                     AS count
FROM enrollment
WHERE campaign_id = ?
GROUP BY minute
ORDER BY minute ASC;
```

**索引保证**：`idx_enrollment_enrolled_at` 使 GROUP BY 按分钟聚合时走索引，避免全表扫描。

## 七、枚举值汇总

| 枚举字段 | 所属表 | 可选值 | 对应 PRD 状态 |
|---------|--------|-------|-------------|

| semester_type | semester | FIRST, SECOND | PRD 6.1 |
| semester.status | semester | ACTIVE, INACTIVE | — |
| course.type | course | REQUIRED, ELECTIVE_MAJOR, ELECTIVE_GENERAL | PRD BR-02 |
| offering.status | course_offering | PENDING, APPROVED, REJECTED | PRD 8 开课状态 |

| enrollment.status | enrollment | ENROLLED, APPROVED, REJECTED, DROPPED | PRD 8 学生选课状态 |

> **enrollment.status 的 DROPPED 状态**：虽然在 V1.0 退课使用 DELETE，但在未来需要退课记录时启用。当前保留在枚举定义中，预留扩展位。

## 八、DDL 完整语句

```sql
-- ============================================
-- CAS V1.0 数据库初始化脚本
-- 目标：MySQL 8.0+
-- 字符集：utf8mb4
-- ============================================

CREATE DATABASE IF NOT EXISTS cas
    DEFAULT CHARACTER SET utf8mb4
    COLLATE utf8mb4_unicode_ci;

USE cas;

-- -------------------------------------------
-- 1. 用户表
-- -------------------------------------------
CREATE TABLE sys_user (
    id           BIGINT       AUTO_INCREMENT PRIMARY KEY,
    username     VARCHAR(50)  NOT NULL,
    password     VARCHAR(255) NOT NULL COMMENT 'BCrypt encrypted',
    real_name    VARCHAR(50)  NOT NULL,
    role         VARCHAR(20)  NOT NULL COMMENT 'STUDENT / TEACHER / ADMIN',
    student_id   VARCHAR(20)  NULL     COMMENT 'student number, only for STUDENT role',
    department   VARCHAR(100) NULL     COMMENT 'college or department',
    major        VARCHAR(100) NULL     COMMENT 'major',
    grade        VARCHAR(20)  NULL     COMMENT 'grade, e.g. 2023级',
    email        VARCHAR(100) NULL,
    phone        VARCHAR(20)  NULL,
    avatar_url   VARCHAR(255) NULL,
    status       TINYINT      NOT NULL DEFAULT 1 COMMENT '1=enabled 0=disabled',
    created_at   DATETIME     NOT NULL,
    updated_at   DATETIME     NOT NULL,

    CONSTRAINT uq_sys_user_username   UNIQUE (username),
    CONSTRAINT uq_sys_user_student_id UNIQUE (student_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE INDEX idx_sys_user_role ON sys_user(role);

-- -------------------------------------------
-- 2. 学期表
-- -------------------------------------------
CREATE TABLE semester (
    id             BIGINT      AUTO_INCREMENT PRIMARY KEY,
    name           VARCHAR(100) NOT NULL COMMENT 'e.g. 2025-2026第一学期',
    academic_year  VARCHAR(20)  NOT NULL COMMENT 'e.g. 2025-2026',
    semester_type  VARCHAR(10)  NOT NULL COMMENT 'FIRST / SECOND',
    start_date     DATE         NOT NULL,
    end_date       DATE         NOT NULL,
    status         VARCHAR(10)  NOT NULL DEFAULT 'INACTIVE' COMMENT 'ACTIVE / INACTIVE',
    created_at     DATETIME     NOT NULL,
    updated_at     DATETIME     NOT NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE INDEX idx_semester_status ON semester(status);

-- -------------------------------------------
-- 3. 课程基础信息表
-- -------------------------------------------
CREATE TABLE course (
    id          BIGINT       AUTO_INCREMENT PRIMARY KEY,
    code        VARCHAR(20)  NOT NULL COMMENT 'course code, e.g. GE001',
    name        VARCHAR(100) NOT NULL COMMENT 'course name',
    type        VARCHAR(20)  NOT NULL COMMENT 'REQUIRED / ELECTIVE_MAJOR / ELECTIVE_GENERAL',
    credits     DECIMAL(3,1) NOT NULL COMMENT 'credits',
    hours       INT          NOT NULL COMMENT 'total hours',
    description TEXT         NULL     COMMENT 'course description',
    created_at  DATETIME     NOT NULL,
    updated_at  DATETIME     NOT NULL,

    CONSTRAINT uq_course_code UNIQUE (code)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE INDEX idx_course_type ON course(type);

-- -------------------------------------------
-- 4. 开课记录表
-- -------------------------------------------
CREATE TABLE course_offering (
    id              BIGINT       AUTO_INCREMENT PRIMARY KEY,
    semester_id     BIGINT       NOT NULL,
    course_id       BIGINT       NOT NULL,
    teacher_id      BIGINT       NOT NULL,
    max_capacity    INT          NOT NULL COMMENT 'max students, e.g. 50',
    min_enrollment  INT          NOT NULL COMMENT 'min students to proceed, e.g. 20',
    enrolled_count  INT          NOT NULL DEFAULT 0 COMMENT 'current enrollment count (denormalized)',
    open_grade      VARCHAR(50)  NULL     COMMENT 'open grade, NULL=all grades',
    open_major      VARCHAR(100) NULL     COMMENT 'open major, NULL=all majors',
    location        VARCHAR(100) NULL     COMMENT 'classroom or location',
    schedule        VARCHAR(100) NULL     COMMENT 'schedule, e.g. 周三7-8节',
    status          VARCHAR(15)  NOT NULL DEFAULT 'PENDING' COMMENT 'PENDING / APPROVED / REJECTED',
    created_at      DATETIME     NOT NULL,
    updated_at      DATETIME     NOT NULL,

    CONSTRAINT fk_offering_semester FOREIGN KEY (semester_id) REFERENCES semester(id),
    CONSTRAINT fk_offering_course   FOREIGN KEY (course_id)   REFERENCES course(id),
    CONSTRAINT fk_offering_teacher  FOREIGN KEY (teacher_id)  REFERENCES sys_user(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE INDEX idx_offering_semester        ON course_offering(semester_id);
CREATE INDEX idx_offering_teacher         ON course_offering(teacher_id);
CREATE INDEX idx_offering_status          ON course_offering(status);
CREATE INDEX idx_offering_semester_status ON course_offering(semester_id, status);

-- -------------------------------------------
-- 5. 选课活动表
-- -------------------------------------------
CREATE TABLE selection_campaign (
    id          BIGINT       AUTO_INCREMENT PRIMARY KEY,
    name        VARCHAR(100) NOT NULL COMMENT 'campaign name',
    semester_id BIGINT       NOT NULL,
    start_time  DATETIME     NOT NULL COMMENT 'start time (精确到分钟)',
    end_time    DATETIME     NOT NULL COMMENT 'end time',
    status      VARCHAR(10)  NOT NULL DEFAULT 'PENDING' COMMENT 'PENDING / ACTIVE / ENDED',
    created_at  DATETIME     NOT NULL,
    updated_at  DATETIME     NOT NULL,

    CONSTRAINT fk_campaign_semester FOREIGN KEY (semester_id) REFERENCES semester(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE INDEX idx_campaign_semester    ON selection_campaign(semester_id);
CREATE INDEX idx_campaign_status      ON selection_campaign(status);
CREATE INDEX idx_campaign_time_range  ON selection_campaign(start_time, end_time);

-- -------------------------------------------
-- 6. 选课记录表
-- -------------------------------------------
CREATE TABLE enrollment (
    id          BIGINT      AUTO_INCREMENT PRIMARY KEY,
    campaign_id BIGINT      NOT NULL,
    offering_id BIGINT      NOT NULL,
    student_id  BIGINT      NOT NULL,
    status      VARCHAR(10) NOT NULL DEFAULT 'ENROLLED' COMMENT 'ENROLLED / APPROVED / REJECTED / DROPPED',
    enrolled_at DATETIME    NOT NULL COMMENT 'enrollment timestamp',
    reviewed_at DATETIME    NULL     COMMENT 'review timestamp',
    created_at  DATETIME    NOT NULL,
    updated_at  DATETIME    NOT NULL,

    CONSTRAINT fk_enrollment_campaign FOREIGN KEY (campaign_id)  REFERENCES selection_campaign(id),
    CONSTRAINT fk_enrollment_offering FOREIGN KEY (offering_id)  REFERENCES course_offering(id),
    CONSTRAINT fk_enrollment_student  FOREIGN KEY (student_id)   REFERENCES sys_user(id),

    CONSTRAINT uq_enrollment_campaign_student UNIQUE (campaign_id, student_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE INDEX idx_enrollment_offering    ON enrollment(offering_id);
CREATE INDEX idx_enrollment_student     ON enrollment(student_id);
CREATE INDEX idx_enrollment_enrolled_at  ON enrollment(enrolled_at);
CREATE INDEX idx_enrollment_status      ON enrollment(status);

-- -------------------------------------------
-- 7. 通知表
-- -------------------------------------------
CREATE TABLE notification (
    id          BIGINT       AUTO_INCREMENT PRIMARY KEY,
    user_id     BIGINT       NOT NULL,
    type        VARCHAR(10)  NOT NULL COMMENT 'APPROVED / REJECTED / SYSTEM',
    title       VARCHAR(200) NOT NULL COMMENT 'notification title',
    content     TEXT         NULL     COMMENT 'notification content',
    course_name VARCHAR(100) NULL     COMMENT 'related course name',
    is_read     TINYINT      NOT NULL DEFAULT 0 COMMENT '0=unread 1=read',
    created_at  DATETIME     NOT NULL,

    CONSTRAINT fk_notification_user FOREIGN KEY (user_id) REFERENCES sys_user(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE INDEX idx_notification_user_read   ON notification(user_id, is_read);
CREATE INDEX idx_notification_created_at  ON notification(created_at);
```
