## 一、技术选型

### 1.1 框架与核心库

| 层 | 技术 | 版本 | 选型理由 |
|------|------|------|---------|
| 构建工具 | Vite | 6.x | 极速 HMR，Vue 3 官方推荐 |
| 前端框架 | Vue | 3.x (Composition API) | 你选择的 |
| 语言 | TypeScript | 5.x | 严格类型约束，对接后端 API 时减少沟通成本 |
| 路由 | Vue Router | 4.x | Vue 官方路由，支持嵌套布局、导航守卫 |
| 样式 | TailwindCSS | v4 | 零运行时、设计 Token 统一管理 |
| UI 组件库 | shadcn-vue | latest | 源码级自定义，与 shadcn/ui 设计理念一致 |
| 图标 | lucide-vue-next | latest | 与 shadcn-vue 生态绑定，树摇友好 |
| 状态管理 | Pinia | 2.x | Vue 官方状态管理，TypeScript 友好 |
| 数据请求 | Vue Query | latest | 自动缓存/失效/重试，覆盖列表页、详情页完整生命周期 |
| HTTP 客户端 | Axios | latest | 拦截器机制适合统一处理 Token 刷新、错误码 |
| 图表 | Apache ECharts + vue-echarts | 5.x | 前端设计方案.md 指定，满足 Dashboard 热门课程与抢课趋势 |

### 1.2 UI 组件库选型分析

**选择 shadcn-vue 的理由：**

| 对比维度 | shadcn-vue | Naive UI | Element Plus |
|---------|-----------|----------|-------------|
| 设计理念 | 源码级拥有，完全自定义 | 黑盒组件 | 黑盒组件 |
| 与 TailwindCSS 关系 | 原生绑定 | 需覆盖样式 | 需覆盖样式 |
| 视觉风格 | Modern SaaS（Linear/Vercel 风） | 现代但不完全匹配 | 传统后台风 |
| 包体积 | 按需引入源码 | 按需加载 | 按需加载 |
| 与设计文档匹配度 | ✅ 完全匹配 | ⚠️ 部分匹配 | ❌ 风格不匹配 |

**结论：** 前端设计方案.md 明确指向 "Linear + Vercel + shadcn/ui" 的 Modern Education SaaS 方向。shadcn-vue 是 Vue 生态中唯一继承这套设计理念的方案，卡片化、简约、高视觉品质。

#### 路由设计原则

1. **角色前缀隔离**：`/student/*` `/teacher/*` `/admin/*` 三套路由互不交叉，URL 即表明身份
2. **Vue Router 懒加载**：每个页面使用动态 `import()`，Vite 自动代码分割
3. **V2.0 扩展预留**：新增模块（成绩管理 → `student/grades/`、课表管理 → `student/schedule/`）直接在对应目录下新建文件夹，不影响现有路由

## 三、核心组件设计规范

### 3.1 组件设计要求

```
1. 覆盖 4 种状态：loading → empty → error → success
2. Props 使用 TypeScript 接口定义，通过 defineProps 接收
3. 支持 class 透传（Vue 默认 class 继承）
4. 每个组件附带类型导出
```

### 3.2 StatusBadge

**用途**：跨页面统一展示业务状态

```vue
<script setup lang="ts">
import type { CampaignStatus, OfferingStatus, EnrollmentStatus } from '@/types'

type BadgeStatus = CampaignStatus | OfferingStatus | EnrollmentStatus

interface Props {
  status: BadgeStatus
  type?: 'campaign' | 'offering' | 'enrollment'
  class?: string
}
</script>
```

**状态 → 颜色映射规则**：

| 状态值 | 显示文字 | Badge 颜色 | 对应场景 |
|--------|---------|-----------|---------|
| `ACTIVE` | 进行中 | blue (primary) | 选课活动 |
| `PENDING` | 未开始 | gray | 选课活动/开课 |
| `ENDED` | 已结束 | gray | 选课活动 |
| `ENROLLED` | 已报名 | blue | 选课记录 |
| `APPROVED` | 审核通过 | green (success) | 开课/选课记录 |
| `REJECTED` | 审核不通过 | red (danger) | 开课/选课记录 |
| `FULL` | 已满员 | red (danger) | 课程容量 |

映射逻辑由 `type` 参数选择对应映射表，确保同一个字符串在不同场景下显示正确的颜色。

### 3.3 CourseCard

**用途**：课程大厅与学生首页的课程展示单元

```vue
<script setup lang="ts">
```


**响应式**：`grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6`

**Hover 效果**：`hover:-translate-y-0.5 hover:shadow-md transition-all duration-200`

### 3.4 EnrollmentButton

**用途**：抢课操作按钮，自动根据 4 个条件决定状态

```vue
<script setup lang="ts">
import type { CampaignStatus } from '@/types'

interface Props {
  campaignStatus: CampaignStatus    // 选课活动状态
  seatsRemaining: number            // 剩余名额
  hasEnrolled: boolean              // 是否已选课
  loading?: boolean                 // 抢课请求中
  onEnroll?: () => void
  class?: string
}
</script>
```

**状态决策表**：

| campaignStatus | seatsRemaining | hasEnrolled | 按钮显示 | 可点击 | 颜色 |
|---------------|---------------|-------------|---------|-------|------|
| `PENDING` | — | — | 未开始 | ❌ | gray |
| `ACTIVE` | > 0 | false | 立即抢课 | ✅ | blue |
| `ACTIVE` | ≤ 0 | false | 已满员 | ❌ | red |
| `ACTIVE` | — | true | ✓ 已选课 | ❌ | green |
| `ENDED` | — | false | 已结束 | ❌ | gray |
| `ENDED` | — | true | 已选课 | ❌ | green |

**交互细节**：
- 点击后立即进入 loading 状态，按钮文字变为「抢课中…」
- 成功 → Toast "抢课成功" + emit 事件更新父组件
- 失败 → Toast 显示错误原因 + 恢复可点击

### 3.5 CountdownCard

**用途**：首页展示选课活动倒计时

```vue
<script setup lang="ts">
import type { CampaignStatus } from '@/types'

interface Props {
  campaignName: string
  status: CampaignStatus
  endTime?: string               // 结束时间（活动进行中）
  startTime?: string             // 开始时间（活动未开始）
  class?: string
}
</script>
```

**状态展示**：

| 状态 | 显示内容 |
|------|---------|
| 未开始 | "距离开始：2天4小时32分钟" |
| 进行中 | "距离结束：12小时20分钟" |
| 已结束 | "状态：已结束" |

- 数字部分用 `font-mono` 等宽字体
- 秒数通过 `useCountdown` composable 实时更新

### 3.6 KPIStatCard

**用途**：管理员 Dashboard 的统计数字卡片

```vue
<script setup lang="ts">
interface Props {

  trendValue?: string              // 趋势文本

```

**布局**：

```
┌────────────────────┐
│ 📚  总课程数         │
│    128              │   ← text-3xl font-bold
│    ↑ 较上月 +5      │   ← text-sm text-success
└────────────────────┘
```

### 6.3 Vue Query 使用规范

```typescript
// composables/useCampaign.ts
import { useQuery } from '@tanstack/vue-query'
import { campaignService } from '@/services/campaign'

export function useCurrentCampaign() {
  return useQuery({
    queryKey: ['campaign', 'current'],
    queryFn: () => campaignService.getCurrent(),
    refetchInterval: 30_000,
  })
}

// composables/useEnrollment.ts
import { useMutation, useQueryClient } from '@tanstack/vue-query'
import { enrollmentService } from '@/services/enrollment'

export function useEnroll() {
  const queryClient = useQueryClient()

  return useMutation({
    mutationFn: (offeringId: number) => enrollmentService.enroll(offeringId),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['courses'] })
      queryClient.invalidateQueries({ queryKey: ['enrollments'] })
    },
  })
}
```

**规范**：
- queryKey 以资源名开头：`['courses']`、`['offerings', { id }]`、`['enrollments', 'mine']`
- 创建/更新/删除操作后，invalidate 相关 queryKey
- 轮询仅用于状态类数据（选课活动状态），不用于静态列表

```typescript
router.beforeEach((to, from, next) => {
  const auth = useAuthStore()

  // 未登录 → 登录页
  if (!auth.isAuthenticated && to.path !== '/login') {
    return next('/login')
```

### 8.3 Axios 拦截器

```typescript
// lib/axios.ts
import axios from 'axios'
import { useAuthStore } from '@/stores/auth'

export const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
  timeout: 10_000,
})

api.interceptors.request.use((config) => {
  const token = useAuthStore().token
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

api.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      useAuthStore().logout()
    }
    return Promise.reject(error)
  },
```
